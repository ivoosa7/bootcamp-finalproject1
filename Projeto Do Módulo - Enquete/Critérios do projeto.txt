Crie um repositório (PÚBLICO), não use um repositório existente. 
Crie um GitHub Project para o repositório (PÚBLICO).
Analise as imagens abaixo do projeto para propor as tarefas. Tente estruturar uma ordem de execução para propor as tarefas, criando tarefas bem definidas com objetivos claros.
Criar no mínimo 6 tarefas (issue).
Ao final de cada tarefa fazer um commit.
Não é necessário seguir com exatidão as imagens do projeto, faça os ajustes que você julgar necessário.
Critérios de aprovação do projeto:

Ter um repositório único para o projeto.
Ter o branch main.
Ter o project (quadro Kanban)  público.
Ter pelo menos 6 tarefas.
Ter todas as tarefas fechadas.
Ter pelo menos 6 commits, um para cada tarefa.
Ser responsivo conforme as imagens apresentadas.
Ser parecido com as imagens de referência.
